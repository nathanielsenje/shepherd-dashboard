🎯 THREE WORKFLOWS

1. EXTRACT GUIDELINES 📊
   Upload UI image → Skill analyzes → Markdown guidelines
   Outputs: Design tokens + component specs

2. REVIEW COMPONENTS 🔍
   Provide code + guidelines → Skill checks → Compliance report
   Outputs: Issues found + corrected code

3. GENERATE COMPONENTS 🛠️
   Provide guidelines → Skill generates → Production code
   Outputs: React/Vue/HTML components ready to use

═════════════════════════════════════════════════

📊 OUTPUT FORMAT

Guidelines are markdown files (.md) containing:

✅ Color Palette (primary, semantic, neutral, gradients)
✅ Typography System (fonts, scales, weights, line heights)
✅ Spacing Scale (xs, sm, md, lg, xl, 2xl)
✅ Shadows & Elevation (sm, md, lg, xl levels)
✅ Border Radius Scale (sm, md, lg, full)
✅ Component Sections (all 13 types with full specs)
✅ Design Tokens Reference Table
✅ Implementation Guidelines & Best Practices

See EXAMPLE_EXTRACTED_GUIDELINES.md for complete example
